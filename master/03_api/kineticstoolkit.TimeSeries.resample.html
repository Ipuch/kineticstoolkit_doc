kineticstoolkit.TimeSeries.resample

TimeSeries.resample(new_time, kind='linear', *, fill_value=None, in_place=False)

Resample the TimeSeries.

Warning

The function resample is unstable, which means it may not be tested or settled yet. Please avoid using this function in production code.

Parameters

  • new_time (ndarray) – The new time vector to resample the TimeSeries to.

  • kind (str) – Optional. The interpolation method. This input may take any value supported by scipy.interpolate.interp1d, such as ‘linear’, ‘nearest’, ‘zero’, ‘slinear’, ‘quadratic’, ‘cubic’, ‘previous’, ‘next’. Additionally, kind can be ‘pchip’.

  • fill_value (Union[ndarray, str, None]) – Optional. The fill value to use if new_time vector contains point outside the current TimeSeries’ time vector. Use ‘extrapolate’ to extrapolate.

  • in_place (bool) – Optional. True to modify and return the original TimeSeries. False to return a modified copy of the TimeSeries while leaving the original TimeSeries intact. The default is False.

Returns

The TimeSeries with a new sample rate.

Return type

TimeSeries

Example

>>> ts = ktk.TimeSeries(time=np.arange(6.))
>>> ts.data['data'] = ts.time ** 2
>>> ts.data['data_with_nans'] = ts.time ** 2
>>> ts.data['data_with_nans'][3] = np.nan
>>> ts.time
array([0., 1., 2., 3., 4., 5.])
>>> ts.data['data']
array([ 0.,  1.,  4.,  9., 16., 25.])
>>> ts.data['data_with_nans']
array([ 0.,  1.,  4., nan, 16., 25.])

>>> ts = ts.resample(np.arange(0, 5.5, 0.5))

>>> ts.time
array([0. , 0.5, 1. , 1.5, 2. , 2.5, 3. , 3.5, 4. , 4.5, 5. ])

>>> ts.data['data']
array([ 0. ,  0.5,  1. ,  2.5,  4. ,  6.5,  9. , 12.5, 16. , 20.5, 25. ])

>>> ts.data['data_with_nans']
array([ 0. ,  0.5,  1. ,  2.5,  4. ,  nan,  nan,  nan, 16. , 20.5, 25. ])
